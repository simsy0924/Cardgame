# HandBattle 공통 수동 테스트 체크리스트

17단계 목적: 테마를 새 엔진으로 이식한 뒤, 같은 기준으로 계속 검증하기 위한 공통 체크리스트입니다.

## 1. 효과 버튼 / 발동 경로

- [ ] 같은 효과 버튼이 2개 뜨지 않는다.
- [ ] `effectIds`가 있는 카드는 신엔진 버튼만 뜬다.
- [ ] `effectIds`가 없는 카드는 레거시 버튼이 유지된다.
- [ ] `continuous`, `procedure`, `replacement`, `processingNegate` 효과는 일반 효과 버튼으로 뜨지 않는다.
- [ ] 필드 존 카드를 클릭하면 필드 존에서 발동 가능한 효과 버튼이 뜬다.
- [ ] 필드 카드의 발동 시 처리 전용 효과는 필드 존 버튼으로 뜨지 않는다.

## 2. 발동 가능성 / 처리 가능성

- [ ] `condition`이 false인 효과는 버튼/후보로 뜨지 않는다.
- [ ] `canResolve`가 false인 효과는 버튼/후보로 뜨지 않는다.
- [ ] 서치 대상이 없으면 서치 효과는 발동할 수 없다.
- [ ] 소환할 몬스터 존이 없으면 소환 효과는 발동할 수 없다.
- [ ] 버릴 패가 없으면 패 버림이 필요한 효과는 발동할 수 없다.
- [ ] 묘지/제외/덱/키덱 대상이 없으면 해당 존을 참조하는 효과는 발동할 수 없다.
- [ ] 내성 때문에 처리 가능한 상대 몬스터가 없으면 제거 효과는 발동할 수 없다.

## 3. 체인 / 코스트 / 해결

- [ ] 발동형 효과는 체인 블록을 만든다.
- [ ] 퀵 효과는 허용된 타이밍에 체인 응답으로 올라갈 수 있다.
- [ ] 유발 효과는 이벤트 후 후보로 올라간다.
- [ ] 코스트는 체인에 올라가기 전에 먼저 지불된다.
- [ ] 효과가 무효되어도 지불한 코스트는 되돌아가지 않는다.
- [ ] 체인은 마지막 링크부터 해결된다.
- [ ] 처리 시 무효는 별도 체인을 만들지 않고 원래 링크의 `resolve`만 스킵한다.

## 4. 강제 / 임의 / 지속 / 소환 절차

- [ ] 강제 유발 효과는 조건 충족 시 자동으로 체인에 올라간다.
- [ ] 임의 유발 효과는 선택창에 뜨고 자동 발동하지 않는다.
- [ ] 지속 효과는 버튼으로 뜨지 않고 조건 충족 중 자동 적용된다.
- [ ] 소환 절차는 체인 블록을 만들지 않는다.
- [ ] 소환 절차는 자신 전개 단계 + 체인 없음 조건을 지킨다.
- [ ] 대체 효과는 체인 블록을 만들지 않는다.

## 5. 필드 카드

- [ ] 패에서 필드 카드를 발동하면 먼저 필드 존에 놓인다.
- [ ] 기존 필드 존 카드가 있으면 묘지로 간다.
- [ ] 필드 카드 발동이 무효되면 방금 놓은 필드 카드는 묘지로 간다.
- [ ] 효과로 필드 존에 놓은 카드는 발동 시 효과 처리를 하지 않는다.
- [ ] 필드 존 지속 효과는 필드 존에 존재하는 동안만 적용된다.

## 6. 네트워크 / AI

- [ ] 실제 상태 변경은 권위 클라이언트에서 한 번만 일어난다.
- [ ] 상대 클라이언트는 같은 효과를 다시 해결하지 않고 상태/diff만 반영한다.
- [ ] 양쪽 화면에서 필드/묘지/제외 수가 같다.
- [ ] AI전에서도 같은 상태 변경이 두 번 일어나지 않는다.
- [ ] 화면이 불필요하게 깜빡이지 않는다.
